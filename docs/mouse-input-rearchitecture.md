# Moonlight macOS 鼠标输入综合重构任务书

## 1. 背景

当前 `moonlight-macos` 的鼠标输入链路已经积累了多轮修补，尤其是在以下方面出现了明显问题：

- 物理滚轮、触控板、Magic Mouse、`GCMouse`、`NSEvent`、`CGEvent` 字段被混合在同一套启发式中处理。
- `hasPreciseScrollingDeltas` 被过度当作“设备类型”信号，而 Apple 官方语义更接近“精度可用性”。
- 去重、量化、trace 锁定、AppKit 抑制逻辑互相叠加，导致：
  - 一次物理滚轮可能被放大成远端多次滚动；
  - 连续滚动可能又被压扁成一次滚动；
  - 鼠标与触控板切换后，方向、状态、连续性互相污染；
  - 延迟与“跟手感”严重不稳定。
- 现有设置中的 `HID` / `MFi` 驱动切换已经无法表达新的输入策略，也不利于默认自动优化。

同时，新的需求已经明确：

- `#6` 期望在较新 macOS 上引入 `CoreHID`，作为新一代 HID 输入路径。
- `#7` 已给出 `CoreHID` 的初步验证型 PR，但该 PR 只适合作为参考样本，不能直接等同于最终方案。
- 本次不是“只修滚轮”或“只加 CoreHID”，而是一次 **鼠标移动、滚轮、触控板、参数调节、设置区域** 的综合重构。

## 2. 本次任务目标

### 总目标

建立一套新的 macOS 输入架构，使 Moonlight 在默认配置下自动选择最优方案，并在高级设置中允许用户按需覆盖。

### 具体目标

- 将输入系统从“驱动名导向”改为“能力与语义导向”。
- 在 `macOS 15+` 引入 `CoreHID` 作为优先的低延迟鼠标相对移动路径。
- 保持 `macOS 12–14` 完整可用，并提供清晰的自动回退。
- 重构滚轮语义，优先恢复 **原生、低延迟、可靠** 的物理滚轮体验，而不是默认做触控板化平滑。
- 保留触控板 / Magic Mouse 的连续高精度滚动语义，不让它们被物理滚轮逻辑污染。
- 重构设置中的输入区域，使其默认全自动，但允许用户做高级调优。
- 为后续的设备增强能力、远端光标辅助、主机兼容调优预留可扩展架构。

## 3. 设计原则

### 3.1 自动优先

- 默认配置必须尽可能做到“开箱即用、自动最优”。
- 用户不应该先理解 `HID`、`MFi`、`CoreHID` 的差异，才能获得正常体验。
- 手动选项用于高级调试、兼容性回退、细粒度偏好控制，不是默认前提。

### 3.2 能力分层，不做品牌分叉

- 输入系统按“能力”建模，而不是按“品牌”硬编码。
- 基础能力至少包括：
  - 相对鼠标移动
  - 垂直滚轮
  - 水平滚轮
  - 连续滚动手势
  - 额外鼠标按钮
  - 可选的高回报率/高频事件流
- `G304` 和 `MX Master 3S` 可以走同一套基础能力层。
- 如果后续引入 `HID++` 或其他厂商协议，也必须作为“可插拔增强层”进入，而不是改成 Logitech 特供方案。

### 3.3 原生滚轮优先

- 普通物理滚轮的默认目标是：
  - 第一格立即响应；
  - 连续 ratchet 连滚时保持稳定节奏；
  - 不出现夸大、回弹、方向漂移；
  - 不默认模拟触控板惯性。
- 若未来提供“平滑滚轮模式”，它必须是高级可选项，而不是默认语义。

### 3.4 连续设备与离散设备分开建模

- 触控板 / Magic Mouse / 连续滚动设备：保留连续、高精度、带 phase / momentum 的语义。
- 普通物理滚轮：保留离散、单调、低延迟语义。
- 外部工具改写过的事件：单独识别与隔离，避免污染设备分类。

### 3.5 CoreHID 是增强路径，不是单点依赖

- `CoreHID` 仅在 `macOS 15+` 可用。
- 本项目最低支持版本保持 `macOS 12`。
- 因此 `CoreHID` 必须设计成：
  - 自动启用的增强路径；
  - 可观测、可回退、可禁用；
  - 不可用时不影响整条输入链路。

## 4. 参考依据与借鉴范围

### Apple 官方

- `NSResponder.scrollWheel(with:)`
- `NSEvent.scrollingDeltaX / scrollingDeltaY`
- `NSEvent.hasPreciseScrollingDeltas`
- `NSEvent.phase / momentumPhase`
- `CGEvent` scroll fields
- `CoreHID`（`macOS 15+`）

关键结论：

- `hasPreciseScrollingDeltas` 只表示精度，不表示来源类型。
- `phase / momentumPhase` 才是连续滚动手势的重要生命周期信号。
- `CoreHID` 在本地 SDK 中已经是公开 Swift 接口，但仅限 `macOS 15+`。

### Issue / PR

- `#6`：确认 `CoreHID` 作为新代 HID 输入路径是明确需求。
- `#7`：可借鉴以下内容：
  - `CoreHID` 基础接入
  - 输入监控权限检查
  - runtime failure / fallback
  - max report rate 上限
  - trackpad / `NSEvent` fallback

但不直接照搬以下内容：

- 过于初步的设备匹配策略
- 只覆盖 `X / Y` 相对移动而未纳入完整输入语义
- 未完成的低版本兼容与更大范围的设置架构设计

### Mos

借鉴点：

- `CGEventTap` 事件拦截
- 合成事件标记
- 跳过已被平滑/远程改写的事件
- 避免二次处理

不直接照搬：

- 默认将物理滚轮“触控板化”的平滑逻辑
- 以本地 UI 体验为核心的插值/惯性模型

### Mouser

借鉴点：

- 高级鼠标能力建模
- 设备目录与能力映射
- 鼠标增强层与底层 hook 解耦

不直接照搬：

- Logitech 专有产品边界
- 以 HID++ 定义整个输入主路径

### UU 远程

目前能确认的架构级参考：

- 使用低层输入能力而非单纯依赖 AppKit 高层事件
- 鼠标移动与滚动是独立建模的
- 远端光标/本地输入链可能存在特殊辅助机制，以降低主观延迟与增强跟手感

本轮把它作为重要参考方向，但不在没有进一步证据前直接复刻其闭源实现细节。

## 5. 新架构总览

新的输入架构按 5 层拆分：

### 5.1 Source Layer（输入源层）

负责采集，不做最终语义判断。

候选输入源：

- `CoreHID`：`macOS 15+` 相对移动优先源，未来可扩展到滚轮相关 usages
- `NSEvent`：AppKit 事件源，仍承担滚轮、触控板、fallback 等职责
- `CGEvent`：辅助读取字段，增强来源识别
- `GCMouse`：保留兼容，但不再成为滚轮主语义判断依据
- 可选厂商增强层：未来若引入 `HID++`，应作为补充能力源

### 5.2 Capability Layer（能力层）

负责描述设备/事件流的“能做什么”，而不是“它叫什么”。

至少建模以下能力：

- `relativeMotion`
- `discreteVerticalWheel`
- `discreteHorizontalWheel`
- `continuousScrollGesture`
- `extraButtons`
- `highReportRate`
- `vendorEnhancementAvailable`

### 5.3 Semantics Layer（语义层）

负责把输入流归入明确语义路径：

- `relativeMotion`
- `discreteWheel`
- `continuousGestureScroll`
- `horizontalWheel`
- `syntheticOrRewrittenScroll`

该层必须明确：

- 什么情况下是物理滚轮；
- 什么情况下是连续手势；
- 什么情况下是外部工具改写事件；
- 什么情况下要切换到 fallback。

### 5.4 Policy Layer（自动策略层）

负责在不同系统、设备、权限状态下自动选择最优路径。

目标策略：

- `macOS 15+`
  - 默认优先 `CoreHID` 相对移动
  - `NSEvent` 继续负责滚轮与触控板
  - 任一路径异常时，自动启用 fallback
- `macOS 12–14`
  - 继续使用 `NSEvent` / 现有兼容路径
  - 但语义层与滚轮逻辑要使用新的分类方式

### 5.5 Settings Layer（设置层）

彻底替换现有“驱动名切换”的思路。

新设置区域目标：

- 默认模式：`Automatic`
- 高级模式：允许覆盖
  - 鼠标移动策略
  - 滚轮策略
  - 触控板策略
  - `CoreHID` 上限频率
  - 回退策略
  - 调试诊断开关

## 6. 滚轮重构要求

滚轮是本次重构的核心。

### 6.1 必须解决的问题

- 单格物理滚轮被放大为多次远端滚动
- 连续 ratchet 滚动被错误压缩成一次
- 触控板与物理滚轮相互污染
- 第一次滚动延迟大
- 使用第三方鼠标工具后事件被改写，导致分类失真

### 6.2 实现要求

- 不再以时间窗去重作为滚轮主逻辑。
- 去重仅允许用于极窄的防抖，而不是滚轮语义判断。
- 物理滚轮采用：
  - 首事件立即派发
  - 同方向同动作可有限聚合
  - 不允许长尾回弹被当作额外 notch
- 连续滚动设备采用：
  - 高精度累计
  - phase / momentum 感知
  - 与物理滚轮完全独立的状态机
- `hasPreciseScrollingDeltas` 不再单独决定“是否量化”。

### 6.3 默认体验要求

- 普通鼠标的滚轮要“原生、稳定、低延迟”。
- 触控板要“连续、平滑、方向正确、不会失效”。
- 切换输入设备后，状态不会串线。

## 7. CoreHID 任务要求

### 7.1 本轮纳入范围

- `macOS 15+` 的 `CoreHID` 鼠标相对移动路径
- 权限检查、失败提示、自动回退
- report rate 上限设置
- 与旧路径并行存在时的健康探测

### 7.2 本轮不直接承诺的范围

- 把全部滚轮逻辑立即迁移到 `CoreHID`
- 把所有厂商扩展按钮都纳入 `CoreHID`
- 在 `macOS 12–14` 模拟 `CoreHID`

### 7.3 实现原则

- 参考 `#7`，但必须自行按官方接口重构实现。
- 优先按 Apple 官方公开接口、SDK 可用性与运行期能力做设计。
- 任何 `CoreHID` 路径都必须：
  - 可关闭
  - 可回退
  - 可诊断

## 8. 设置区域重构要求

新的输入设置区不再围绕旧的“驱动”概念组织，而是采用与用户心智一致的一级结构：

- 鼠标
- 快捷键
- 手柄

其中滚轮、触控板、CoreHID 状态与绝大多数手感参数都属于 **鼠标一级区域**。

### 8.1 默认层

- `Automatic`
- 使用推荐策略
- 尽量不给用户暴露底层术语
- 在 `macOS 15+` 上默认自动优先 `CoreHID`，不可用时自动回退

### 8.2 鼠标一级直接暴露

普通用户常改、且会显著影响手感的内容直接放在鼠标一级：

- Mouse Input Strategy
- Mouse Mode
- Pointer Speed
- Physical Wheel Mode
- Physical Wheel Distance
- Enhanced Wheel Mode
- Enhanced Wheel Distance
- Gesture / Trackpad Distance
- Reverse Mouse Scrolling Direction

### 8.3 鼠标内高级层

不需要普通用户频繁操作、但排障和精调很重要的内容放进鼠标区域内的折叠高级设置：

- `CoreHID Max Mouse Report Rate`
- 鼠标左右键交换
- Touchscreen Mode

### 8.4 手柄区域

手柄与鼠标平级，保留独立一级区域。常用项直接显示；兼容/排障项收进手柄区域内部的高级设置。

直接显示：

- Multi-Controller Mode
- Rumble Controller

手柄内高级设置：

- Controller Driver
- Swap A/B and X/Y Buttons
- Emulate Guide Button
- Gamepad Mouse Emulation

### 8.5 设计要求

- 简单模式明确、默认安全
- 高级模式可诊断
- 说明文案必须能解释“当前实际使用的是哪条路径”

## 9. 远端光标辅助与低延迟体验

基于对 `UU` 的观察，后续需要把“远端光标辅助”列为独立研究工作流。

### 本轮结论

- 可以将其纳入架构预留，但不作为本轮阻塞项。
- 需要在文档中明确保留以下扩展位：
  - 远端光标位置反馈
  - 本地光标 / 远端光标协同策略
  - 独立输入链路优化

### 本轮要求

- 不得在当前重构中把未来的光标辅助路径堵死。
- 日后如果验证该方向有效，应能在新设置区中作为高级增强选项出现。

### 本轮实现边界

- 本轮先交付 **可正式测试的第一版**，不是一次性把所有输入体验做到最终形态。
- `CoreHID` 本轮先覆盖 **鼠标相对移动**，滚轮仍以新的 AppKit / CGEvent 语义路径为主。
- 设置区本轮已从“驱动导向”重构为“鼠标 / 快捷键 / 手柄”一级分区；滚轮与触控板调速直接进入鼠标一级，`CoreHID Max Mouse Report Rate` 等精调项进入鼠标内高级设置。

### Phase 1：诊断与分类清理

- 统一输入诊断字段
- 区分物理滚轮、连续滚动、外部改写事件
- 清理旧的 trace / dedup 残留状态

### Phase 2：滚轮语义重构

- 建立新的离散滚轮状态机
- 建立新的连续滚动状态机
- 修复物理滚轮首响应与连续滚动压扁问题

### Phase 3：CoreHID 鼠标移动接入

- 引入 `CoreHID` 自动优先路径
- 完成权限、失败通知、fallback
- 增加 report rate 上限

### Phase 4：设置区域重构

- 重新设计鼠标/滚轮/触控板设置结构
- 以“自动 + 高级覆盖”替代旧驱动命名

### Phase 5：扩展与验证

- 更广泛验证 `G304`、`MX Master 3S`、触控板、Magic Mouse
- 与 Sunshine / Apollo / Foundation-Sunshine 做主机侧兼容性核对
- 继续研究 `UU` 的远端光标辅助方向

## 11. 验收标准

### 物理滚轮

- 单格滚动不会被放大为多次远端滚动
- 连续滚动不会被压扁成一次
- 上下方向都能稳定工作
- 首次滚动响应明显改善

### 触控板 / Magic Mouse

- 连续滚动正常
- 惯性正常
- 切换到鼠标后再切回，不会方向错乱或失效

### CoreHID

- `macOS 15+` 可自动启用
- 无权限时有清晰提示且能自动回退
- 低版本不崩溃、不失效

### 设置与诊断

- 用户能知道当前实际路径
- 自动模式对多数用户足够好用
- 高级设置能支撑排障与体验微调

## 12. 正式测试清单

### 环境准备

- 测试机至少准备：
  - 一只普通滚轮鼠标（如 `G304`）
  - 一只高端鼠标（如 `MX Master 3S`，若有）
  - Mac 触控板或 Magic Mouse
- `macOS 15+` 机器需要至少测一次 `Automatic` 与 `CoreHID`
- 测试前尽量关闭 `Mos`、`BetterMouse`、`LinearMouse`、`Mouser` 等第三方输入改写工具

### 设置入口

- 打开设置中的 `Mouse Input Strategy`
- 重点验证：
  - `Automatic`
  - `Compatibility`
  - `GameController`
  - `CoreHID`（仅 `macOS 15+`）

### 鼠标移动

- 在 `Automatic` 下测试：
  - 相对移动是否跟手
  - 快速甩动是否丢帧或抖动
  - 远程桌面模式下绝对指针同步是否仍正常
- 在 `CoreHID` 下测试：
  - 首次进入串流后是否立即可用
  - 无权限时是否自动回退而不是失效
  - 与 `Automatic` 相比是否有更低的体感延迟

### 物理滚轮

- 单格向上 / 向下各滚动 10 次：
  - 不应出现“一格触发多次远端滚动”
  - 不应出现“只有向上能滚、向下不行”
- 连续快速滚动：
  - 不应被压缩成一次
  - 不应出现明显首包延迟
- 鼠标滚轮后立刻切回触控板：
  - 方向不反转
  - 不失效

### 触控板 / Magic Mouse

- 缓慢连续滚动
- 快速滚动 + 惯性滚动
- Space 切换、全屏切换、鼠标使用后再次滚动
- 观察是否出现：
  - 方向反转
  - 中途卡死
  - 离开 Space 再回来才恢复

### 日志观察点

- 关注是否出现 `CoreHID mouse fallback`
- 若 `Automatic` 下无权限，允许看到 fallback 日志，但不应导致鼠标失效
- 若滚轮仍异常，优先对比：
